# Phantom SDK Wallet Connect and Signs - Inventagious Post

## Overview

Inventagious uses **Phantom Connect SDK** for seamless wallet connection and message signing. The SDK provides multiple authentication methods making it easy for users to connect their wallets - no browser extensions required!

## Key Information

### Wallet Connection Methods

Inventagious supports four easy ways to connect through Phantom SDK:

1. **Google OAuth** - Sign in with Google and get an embedded non-custodial wallet
2. **Apple ID** - Sign in with Apple and get an embedded non-custodial wallet  
3. **Phantom Browser Extension** - Connect with your installed Phantom extension
4. **Mobile Deeplink** - Connect via QR code with Phantom mobile app

### User Experience Benefits

- **No Extension Required** - Users can connect with just Google or Apple sign-in
- **Multiple Options** - Choose the connection method that works best for you
- **Secure Authentication** - Message signing proves wallet ownership securely
- **Mobile Friendly** - Works seamlessly on mobile devices
- **Non-Custodial** - Users always control their own wallets

### How It Works

1. User clicks "Connect Wallet"
2. Chooses connection method (Google, Apple, Extension, or Mobile)
3. Approves connection
4. Signs authentication message to prove ownership
5. Ready to use Inventagious!

### Security Features

- All signatures are verified server-side
- Message signing only for authentication (no transaction permissions)
- OAuth profile data kept private
- Secure token storage

## Notes

- Users can connect without installing browser extensions
- Embedded wallets created through OAuth are non-custodial
- All wallet types support transaction signing and sending
- Mobile users can connect via QR code with Phantom app
